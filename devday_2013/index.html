Puppet vs Chef

Comunidade

Atualmente Puppet tem uma maior user base; Chef é mais utilizado nos Estados Unidos; Chef tem aumentado consideravelmente sua base de usuários nos ultimos anos; PuppetLabs: mais tradicional, localizada em Portland; OpsCode: empresa menor e com beneficios mais agressivos, localizada em Seatle;

Tecnologia

Puppet: graph-based e model-driven environment. Ordem de execução é imposta pelo usuário atravez de dependencias(mais popular com sysadmins) Custom DSL e extensivel com Ruby(focado em simplicidade) Chef: Força a ordem de execução(mais popular com devs) Ruby DSL (focado em automação de cloud)

Configuração

Puppet quickstart (serverless)
$ vim test.pp


file { 'testfile-puppet':
  path    => '/tmp/testfile-puppet',
  ensure  => present,
  mode    => '0640',
  content => "I'm a test file.\n"
}

$ puppet apply test.pp
Chef quickstart (solo)

            
Chef quickstart (solo)
$ knife cookbook create test
$ vim test/recipes/default.rb

  file "/tmp/testfile-chef" do
    mode    "0640"
    content "I'm a test file.\n"
    action  :create
  end

$ chef-solo -o test -N test
